## Project Overview

- **Type:** Fullstack (Flask REST backend + Next.js frontend)
- **Domain:** AI-era team collaboration & R&D management — Agents are first-class executors; requirements/BUGs can be assigned to humans **or** AI Agents (dev-agent / qa-agent), with a full mixed-collaboration audit trail.
- **Languages:** Python (~55%, 81 files incl. 27 test modules), TypeScript (~45%, 67 files)
- **Frameworks:** Next.js 14 (App Router) + React 18 + Tailwind + @dnd-kit + SWR · Flask 3 + SQLAlchemy 2 + Flask-JWT-Extended
- **Persistence:** SQLite (`backend/aragon.db`, WAL + `synchronous=NORMAL`, auto-created + seeded on first run)
- **Entry Points:** `backend/app.py` (`create_app`, port 5000) · `frontend/app/page.tsx` (redirect) · `frontend/app/(app)/layout.tsx` (app shell) · `start-dev.ps1` (one-shot dev launcher)
- **Maturity:** 10 feature rounds shipped. Backend **371 pytest cases collected, 0 collection errors**; frontend `tsc --noEmit` clean.

### How the app runs
- **Both at once:** `.\start-dev.ps1` from the repo root (creates `backend\.venv`, installs deps, prepares `.env.local` on first run; `-Install` forces a refresh).
- Backend `:5000` — `cd backend` then `python app.py`. First run: `db.create_all()` → `schema_sync` ALTERs → storage introspection → stale-lock release → `seed_if_empty()`.
- Frontend `:3000` — `cd frontend` then `npm run dev`. Talks to backend via `NEXT_PUBLIC_API_BASE` (default `http://localhost:5000/api`).
- Health: `GET /api/health` → `{status, db, llm:{enabled,provider,model}, storage:{persistent,journal_mode,synchronous,foreign_keys}}`. Deliberately contains **no** filesystem path.
- Seed is **one row per category** (8 rows, all registered in `seed_records`). Seeded login: `admin`. Built-in Agents: `dev-agent`, `qa-agent`.

### Quality gates
- Backend: `cd backend` → `python -m pytest -q`. Judge **relatively**: baseline before changes, then require zero failures and no drop in collected count (currently **371**).
- Frontend: `npm run typecheck` and `npm run build`.

### Agent Runtime & Autopilot (`backend/services/`)
Entry: `agent_runner.py` (129) → `agent_autopilot.py` (254)
- Deterministic orchestration. `agent_runner.advance_one` pushes a ticket one legal step, writes an agent comment and an `actor_type=agent` audit row. `agent_autopilot` layers self-claim, run loops and dev→qa handoff.

### Real Agent Execution — LLM layer (`backend/services/agent_executor.py`, `agent_prompts.py`, `llm/`)
Entry: `backend/services/agent_executor.py` (71)
- Where an agent's comment text comes from: a real LLM when `AGENT_LLM_*` is configured, otherwise the deterministic `AGENT_FORWARD` template. **The LLM only produces content — it never decides a transition.**
- **`agent_executor`:** `generate_work(entity, ticket, agent, to_status, fallback_message) -> str` · `wall_budget_seconds()`. Blanket `except` by design: any LLM failure degrades to the template, never a 5xx. Disabled under `TESTING`; output >20k chars discarded.
- **`agent_prompts`:** `ACTION_BRIEF[(entity, kind, status)]` (keys must match `AGENT_FORWARD` verbatim) · `brief_for(...)` · `build_context(...) -> (system, user)`; recent-comment lookup runs in `no_autoflush` so no SQLite write lock is taken across the network call.
- **`llm/`:** `is_enabled()` · `complete(system, user, …) -> LLMResult` · `describe()` (secret-free health block) · `LLMConfig.from_env()` (frozen; unknown provider → `"none"`) · `AnthropicProvider` / `OpenAICompatProvider` / `get_provider(cfg)`. Stdlib `urllib` only — **zero third-party deps**; only `LLMError(kind ∈ config|http_4xx|http_5xx|timeout|network|parse)` escapes; retries only timeout/5xx/network with 0.5s→1.0s backoff.

### Persistence, Schema Sync & Data Governance (`backend/services/`, `backend/tools/`)
- **`schema_sync.py` (59)** — `ADDITIVE_COLUMNS: [(table, column, DDL)]` + `sync_additive_columns(engine)`. `create_all()` never ALTERs an existing table and there is no Alembic, so **every new model column must be registered here** or existing DBs 500 with `no such column`. **ADD COLUMN only** — the first type/constraint change or backfill means adopting a real migration tool.
- **`persistence.py` (143)** — `describe_storage(uri)` · `log_storage_summary(app)` · `collect_storage_info(app, session)` · `release_stale_agent_locks(session)`. DB path goes to logs only, never to `/api/health`. Lock release is **single-process only** — disable via `RELEASE_STALE_LOCKS_ON_STARTUP=false` under multi-worker gunicorn.
- **`lifecycle.py` (197)** — governance guards: `would_orphan_admins` / `active_admin_count` / `conflict_last_admin` · `project_references` / `conflict_project_has_tickets` · `agent_open_workload` / `conflict_agent_has_open_tickets` · `delete_ticket_cascade` (never commits, never deletes the ticket itself) · `unassign_ticket` (idempotent; never touches `status`/`position`). Conflicts are 409 **without** an `allowed` key, because the frontend routes drag errors on `err.allowed`.
- **`seed.py` (129)** — `seed_if_empty()`. Exactly **8 rows, one per category**, each registered via `SeedRecord.mark(...)` in the same transaction. Tickets are seeded unassigned and in the initial status; the sample comment is authored by `dev-agent` so the "never notify yourself" invariant holds.
- **`tools/purge_demo_data.py` (651)** — offline CLI, **dry-run by default** (`--apply`, `--no-backup`, `--json`, `--database-url`; exit 0/1/2). Deletes only rows that are registered, cascaded by a ticket deletion, or orphaned. **`comments` / `activities` / `notifications` are never trimmed by count** — unregistered rows are presumed real user data. Order-critical startup: set env, *then* `from app import create_app`.
